Analyser for Challenge Caches based on regional (maps, quads, counties) challenges

This macro is designed to process "found cache" data in a GSAK dataset to see how the dataset meets the requirements of a particular challenge cache. Currently the following types of challenge caches can be handled
Challenge Code Challenge Type
DDeLorme Challenge based on mapbooks such as Gazetteer
QQuadrangle based on USGS topo map publications or data, incl ADC maps
EEuropean quads
CCounty challenges
PIrregular polygonal region challenges (Thomas Maps)

Note that type D, C and Q require that every cache in the database have the State provided. The macro will update empty county and state fields in the database.

Note that each challenge is a little bit different even among very similar ones. What they have in common is that various regions (rectangular or polygon-shaped) have to be inspected to see if at least one of the allowed cachetypes have been found there. Some challenges allow virtual caches, some do not, etc. Some allow only caches found after a particular date. Many have additional rules. This macro is designed only for challenges where you must find a cache in various geographic regions.

A NOTE OF CAUTION: DO NOT ASSUME that the macro has the complete rule set implemented for each challenge, particularly for unenforcable rules and ones that are just smoke-screen. In particular, some challenges may be satisfied by placing a cache; that feature is not implemented here as each challenge cache owner has different nuances at different times on this facet; so this processing must be done manually for each case.

C-type Challenges:

County challenges: the user is needs to find a geocache in each of a set of counties within some state or states. For Germany it analyzes the analogous situation: "Kreise" and/or "Kreisstaedte" within states. C-type challenges are ones for which GSAK has the ability to determine in which county a particular geocache is located.

D-type Challenges:

DeLorme Gazetteer challenges (these are always rectangular regions where the boundaries are parallel to lat/lon): All existing DeLorme Challenges which are known to macro author are included.

E-type Challenges installed in this version:

European challenge caches with rectangular regions: These include by country: Netherlands - 6 Dutch Challenge caches "Top25 To Move"

Q-type Challenges (Topo Quads such as USGS topo maps

Quads are quadratic regions similar based on minutes or degrees of latitude and longitude. No mapbook is involved. In the USA there are official quads which have been given names. But quads can also be based on some other regular system. Included are these:
  • 7.5 degree quads
    • CA: SanFrancisco Bay Area Sacramento Valley area LA County area (needs testing) TN (needs testing)
  • 15 minute quads
    • CA: Monterey Bay Area
  • 1 minute quad challenges
    • CA: Davis
  • ADC map books (page borders are parallel to lat/lon
    • Maryland ADC books for: Montgomery and StMary's County

P type Challenges: (any other irregular shaped regions, borders not parallel to lat/lon, etc)

Thomas Map book for: Portland Oregon; Kings County WA.

Configuration

There are a large number of challenge caches that can be processed with this macro. Those which have been programmed into this macro sit in the background available for installtion. Click Configure button to select the ones you want to be able to select from on the main menu.

Modes of Operation

Macro has two modes: (controlled by "Run for NotFound Caches" checkbox)

- to process found caches in a database (leave "Run for NotFound Caches" un-checked)

- to process caches which have NOT been found to see which caches are available Note that you can check as many challenges as appear installed and they will all run.

Output of macro

The results of the processing wild display here after processing is complete. You can also select: Graphical display of summary in Google Earth. You need to have Google Earth installed and associated with kml file types. Selecting file output results in the following
  • CSV - a file that you can open with Excel to do further processing of the results
  • SQL - an text file containing the SQL database of the resulting regions
  • GPX - GSAK GPX file export of all caches in the results

Notes:

This macro works on the entire selected database; any filters are cleared. So be sure the database is up-to-date, that your identity is set correctly in GSAK with Tools, Options, "Method for matching...". If you change that, then be sure to run Database, Repair/Defrag to recalculate the status.

The cache list: any number of caches can be requested; most challenge cache owners want you to submit just ONE. Some early challenge caches required that you to list the coordinates, but most do not. Most expect some sort of bookmark list, either one of the ones that you can set up at geocaching.com or some other list. This macro cannot automate the setting-up of the ones a geocaching.com, but it will give you an html file that should make it easy for you.

You may request a specific number of caches (1, 2, etc) to be returned in the output or enter the word: All Note that "All" may result in taking a very long time to complete if you have a lot of finds; it may also result in a very cluttered display. If you select "1", then your output includes a lot of extra information for the one cache that is displayed for each region for which you have satisfied the challenge. If you select more than one, then only the GCcodes (linked to the cachepage) will be shown,
